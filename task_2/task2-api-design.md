# Задание 2 - Проектирование API

## 1. REST API запрос

Так как нужно только получить список магазинов-партнеров, лучше всего использовать GET-запрос.

### API Endpoint: 
/api/v1/partner-stores 
### Пример полного запроса:
```GET https://api.petrushka-green.ru/api/v1/partner-stores ```

Данный запрос вызывается при открытии пользователем экрана «Магазины партнеров» в мобильном приложении. Для определения ближайшего магазина и корректного расчета времени доставки можно передать город или локацию пользователя в query-параметрах.

### Пример запроса с query-параметрами:
``` GET https://api.petrushka-green.ru/api/v1/partner-stores?city_id=1&lat=55.751244&lon=37.618423 ```

city_id	- идентификатор города пользователя;
lat	- широта пользователя;
lon	- долгота пользователя.

## 2 Ответ API
В ответе сервер возвращает список магазинов-партнеров, которые должны отображаться на экране приложения. Структура успешного ответа состоит из полей "status" и "stores" с типами данных "str" и "array" соответственно. "status" определяет статус ответа API, а "stores" - список магазинов-партнеров. Структура ответа с ошибкой состоит из полей "status" и "message" с типами данных "str", где "message" - сообщение с описанием ошибки.

Исходя из приведенного в задании макета, сервер должен возвращать следующие данные: 
1. идентификатор магазина (тип данных: int)
2. название магазина (тип данных: str)
3. логотип магазина (тип данных: str)
4. тип доставки (тип данных: str)
5. текст доставки (тип данных: str)
6. время доставки (тип данных: str)
7. ссылку для перехода на внешний ресурс при клике на карточку магазина (тип данных: str)

Тип доставки может иметь следующие значения: scheduled - доставка по интервалу времени и fast - быстрая доставка.

Возможные HTTP-статусы ответа:
200 OK - запрос успешно обработан, возвращён список магазинов;
400 Bad Request - некорректные параметры запроса;
404 Not Found - указанный ресурс не найден;
500 Internal Server Error - внутренняя ошибка сервера.

### Пример ответа в JSON при статусе с ошибкой:
Если при обработке запроса возникает ошибка (например, ресурс не найден, параметры запроса переданы некорректно или произошла внутренняя ошибка сервера), API возвращает соответствующий HTTP-статус и сообщение с описанием ошибки. В данном примере приведён ответ при статусе 404 Not Found:

```json
{
  "status": "error",
  "message": "Resource not found"
}
```
